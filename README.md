# Splunk-Lab (Basics)

Project Overview: 

This project demonstrates hands-on experience with Security Information and Event Management (SIEM) operations using Splunk. The objective was to ingest, parse, and analyze security logs within a Splunk environment to simulate tasks commonly performed by a Tier 1 Security Operations Center (SOC) Analyst.

Objectives: 
- Gain hands-on experience using Splunk as a Security Information and Event Management (SIEM) platform.
- Learn how to ingest and manage log data from various sources within a Splunk environment.
- Develop proficiency in using Search Processing Language (SPL) to search, filter, and analyze security events.
- Identify suspicious activities, anomalies, and potential Indicators of Compromise (IOCs) through log analysis.
- Understand the role of log monitoring and event correlation in Security Operations Center (SOC) workflows.
- Strengthen foundational cybersecurity investigation and incident detection skills.
- Apply SOC Level 1 analyst techniques in a simulated enterprise environment.

Lab Environment:

This lab was completed within the TryHackMe SOC Level 1 training environment using a preconfigured Splunk instance. The environment simulated a Security Operations Center (SOC) workflow by providing log data for ingestion, searching, and analysis.

Methodology/Steps Performed

Step 1: Accessing the Splunk Environment

Started the TryHackMe lab machine and accessed the Splunk Enterprise web interface by entering the generated IP address into the Firefox browser within the simulated environment.

Step 2: Uploading Log Data

Uploaded the provided VPN log dataset into Splunk Enterprise by selecting the VPNLogs.json file located in the /root/Rooms/SplunkBasic/ directory. This imported the log data for analysis and investigation.

Step 3: Configuring Data for Analysis

Reviewed and configured the imported data to ensure fields were properly extracted and formatted, allowing for effective searching, filtering, and event analysis within Splunk.

Step 4: Determining the Total Number of Events

The first phase of the investigation was to identify the total number of events contained within the uploaded log file. Using a Splunk search query, I examined the dataset and determined that the log file contained 2,862 events available for analysis.
